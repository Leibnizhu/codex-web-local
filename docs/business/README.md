# Business 文档

## 本页职责
定义业务说明文档的最小结构与更新触发，避免业务知识散落在提交记录中。

## 适用内容
- 交互流程与产品行为说明
- 模块职责与数据流
- 业务约束与兼容性约定

## 当前状态
当前仓库业务说明以根 `README` 为主，本目录预留为后续业务文档入口。

## 更新触发
出现下列变更时补充本目录：
- 新增关键交互流程
- 重要状态机或业务规则变更
- 影响用户行为的功能调整

## 建议模板（最小可读）
新增业务文档时建议至少包含：
- 背景与目标
- 核心流程（输入、处理、输出）
- 边界条件与异常行为
- 验收口径（如何判断行为正确）
